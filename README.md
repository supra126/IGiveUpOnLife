# 不想怒力了 I Give Up On Life

> 打造完整品牌視覺資產？不用，我不想努力了。

一個由 AI 驅動的產品行銷內容設計工具，讓 AI 幫你結合產品識別、品牌故事與競品策略，生成專業的廣告海報概念與視覺素材。

## ✨ 功能特色

### 🎯 Phase 1: 視覺策略選擇（秒速決策）

- **智能產品分析**：上傳產品圖片，AI 自動解讀視覺特徵與品牌語意
- **三條策略路線**：即時生成 3 種截然不同的行銷策略方向（純文字策略卡片）
- **詳細策略資訊**：每條路線包含主打標語、視覺風格描述、目標受眾定位
- **快速響應**：無需等待圖片生成，3 秒內完成策略分析

### 🚀 Phase 2: 完整內容企劃生成（自動觸發）

- **8 張圖完整企劃**：包含 2 張主圖 (1:1) + 6 張社群長圖 (9:16)
- **自動流程**：選擇策略路線後自動規劃完整內容，無需手動觸發
- **競品文案分析**：支援輸入參考文案，AI 拆解說服邏輯並應用於您的產品
- **可編輯腳本**：所有生成的文案與視覺提示詞都可即時編輯調整
- **一鍵生圖**：直接從企劃腳本生成最終視覺素材
- **雙模式切換**：腳本審閱模式 ⇄ 圖片製作模式

### 📊 專業報告輸出

- **策略報告下載**：完整的產品分析、策略選擇與內容企劃 (.txt)
- **視覺素材下載**：所有生成的圖片可單獨下載保存
- **參考圖上傳**：支援為單張圖片上傳參考圖（Logo、顏色等）

## 🛠️ 技術棧

- **前端框架**: React 19 + TypeScript
- **構建工具**: Vite 6
- **AI 模型**: Google Gemini (gemini-2.5-flash, gemini-3-pro-image-preview)
- **樣式**: Tailwind CSS
- **字體**: Inter + Playfair Display

## 📦 安裝與使用

### 環境要求

- Node.js 18+
- npm 或 yarn

### 安裝步驟

```bash
# 克隆專案
git clone https://github.com/mag477/IGiveUpOnLife.git

# 進入專案目錄
cd IGiveUpOnLife

# 安裝依賴
npm install

# 啟動開發伺服器
npm run dev
```

### 配置 API Key

1. 點擊右上角的「API 設定」按鈕
2. 輸入您的 Google Gemini API Key
3. API Key 會安全地儲存在瀏覽器本地存儲中

> 💡 如何獲取 Gemini API Key：前往 [Google AI Studio](https://aistudio.google.com/app/apikey) 註冊並創建 API Key

## 🎨 使用流程

### 方案 A：策略優先流程（當前版本）

1. **一次填完所有資訊**

   - 上傳產品圖片（支援 JPG、PNG）
   - 填寫產品名稱、產品資訊、產品網址（可選）
   - 填寫參考文案 / 競品參考（可選）

2. **AI 策略分析**

   - 點擊「啟動努力」按鈕
   - 等待 AI 分析產品（約 3-5 秒）

3. **選擇視覺策略路線**

   - 查看 3 條不同的策略路線（純文字）
   - 每條路線包含：主打標語、視覺風格、目標受眾
   - 點選一條路線

4. **自動生成 8 張圖腳本**

   - 系統自動進入 Phase 2
   - AI 自動規劃完整的 8 張圖內容腳本
   - 完成後自動滾動到內容區域

5. **腳本審閱與編輯**

   - 切換至「腳本審閱模式」
   - 檢視並編輯所有文案、Prompt、構圖說明

6. **圖片製作**

   - 切換至「圖片製作模式」
   - 逐張生成圖片（可為單張上傳參考圖）
   - 下載所有生成的圖片

7. **匯出報告**
   - 下載完整策略報告 (.txt)

### 回復到舊版本（有 3 張概念海報）

- 參考文案輸入在 Phase 2 區域

## 🌟 核心特點

- **零設計門檻**：不需要設計經驗，AI 自動生成專業視覺策略
- **品牌一致性**：基於品牌故事與產品特性，確保視覺風格統一
- **競品洞察**：可選擇性分析競品文案，學習成功的說服邏輯
- **完全可控**：所有 AI 生成內容都可編輯，保持創意自主權
- **流暢體驗**：精心優化的動畫與 UI，提供愉悅的使用體驗

- **成本優化**：只生成用戶真正需要的圖片，節省 API 成本

## 📋 版本歷史與回復說明

### 當前版本：v2.1 - 策略優先流程 (Plan A)

**主要改進：**

- ✅ Phase 1 簡化為純文字策略卡片（3 秒內完成）
- ✅ 所有輸入資訊前置（參考文案移到初始上傳區）
- ✅ 選擇路線後自動觸發 Phase 2 生成
- ✅ 自動滾動到內容區域
- ✅ 節省 API 成本（不生成未選擇路線的圖片）

**適合情境：**

- 希望快速決定策略方向
- 預算有限，需控制 API 成本
- 更注重最終 8 張圖的品質

### 舊版本：v2.0 - 視覺預覽流程

**特色：**

- Phase 1 為每條路線生成 3 張概念海報（共 9 張）
- 用戶可透過視覺預覽輔助決策
- Phase 2 需手動點擊按鈕觸發
- 參考文案在 Phase 2 區域輸入

**適合情境：**

- 需要視覺預覽輔助策略決策
- 不在意 API 成本
- 希望看到所有路線的視覺效果

**回復指令：**

```bash
# 永久回復到 v2.0
git reset --hard 055f8bf
git push --force  # 如果需要推送到遠端

# 臨時查看 v2.0（不改變當前版本）
git checkout 055f8bf

# 回到最新版本
git checkout main
```

**版本對比：**

| 功能             | v2.0 (舊版)       | v2.1 (當前)  |
| ---------------- | ----------------- | ------------ |
| Phase 1 等待時間 | ~30 秒            | ~3 秒        |
| Phase 1 輸出     | 9 張圖片          | 3 張文字卡片 |
| Phase 2 觸發     | 手動點擊          | 自動生成     |
| 資訊輸入         | 分兩階段          | 一次完成     |
| API 成本         | 較高（多 6 張圖） | 較低         |
| 視覺預覽         | 有                | 無           |
